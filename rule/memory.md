# Memory 记录规则

## 1. 记录目标

memory 只记录稳定、可复用、对后续工作有判断价值的事实，不记录一次性的过程噪音。

## 2. 值得记录的内容

- 已验证的构建命令或构建方式。
- 目标名、语言标准、目录职责这类稳定事实。
- 仓库约定俗成且反复出现的编码习惯。
- 容易再次踩坑的限制条件。
- 已确认有效的排错结论。

## 3. 不要记录的内容

- 临时计划。
- 一次性的任务状态。
- 尚未验证的猜测。
- 很快会过期的中间日志。
- 能直接从源码即时读出的琐碎细节。

## 4. 当前仓库适合沉淀的事实示例

- 当前项目使用 CMake + C++20。
- 当前可执行目标名是 `venthmi`。
- `build/` 是生成目录，不直接编辑。
- `src/main.cpp` 是当前唯一源码入口。

## 5. 记录原则

- 先验证，再记录。
- 尽量短句表达，一个事实一行。
- 发现事实失效时及时修正或删除。
